# Эпик 4 — Конвейер генерации ответов (Response Pipeline)

## Основная информация

| Поле | Значение |
|------|----------|
| ID | E4 |
| Статус | Бэклог |
| Зависит от | E1 (few-shot индекс в Qdrant), E2 (persona.json), E3 (Mem0) |
| Необходим для | E5 (Telegram-бот доставляет черновики из этого pipeline) |

## Описание

Центральный pipeline системы. Запускается при каждом входящем сообщении из whitelist-чата и проходит следующие шаги:

1. **Классификация** — детектирует чувствительные темы (финансы, даты встреч, обещания). Если найдены — черновик помечается `requires_confirmation=True` и никогда не отправляется автоматически.
2. **Few-shot поиск** — ищет в Qdrant top-5 семантически похожих пар «входящее → ответ». Если `score < 0.65` — флаг `low_confidence=True`, уведомление владельцу вместо черновика.
3. **Извлечение памяти** — `mem0.search()` возвращает релевантные факты о собеседнике и пользователе.
4. **Сборка промпта** — объединяет persona + few-shot примеры + контекст диалога + воспоминания в один промпт, укладываясь в лимит токенов.
5. **Вызов LLM** — отправляет промпт в OpenRouter (OpenAI-совместимый API), получает черновик.
6. **Сохранение черновика** — сохраняет в SQLite со статусом `pending`, доставляет в Aiogram-бот.

Устойчивость к сбоям: при недоступности LLM или Qdrant — сообщение попадает в `MessageQueue` (SQLite) и обрабатывается после восстановления. Retry с exponential backoff, max 5 попыток, max ожидание 5 минут.

## Критерий готовности эпика

- [ ] Входящее сообщение проходит все 6 шагов и черновик появляется в Aiogram-боте
- [ ] Чувствительное сообщение (финансы/даты) никогда не отправляется автоматически
- [ ] При недоступности LLM черновик не теряется (попадает в очередь)
- [ ] Время от получения сообщения до появления черновика < 10 секунд (при штатной работе LLM API)
