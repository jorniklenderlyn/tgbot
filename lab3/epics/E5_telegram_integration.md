# Эпик 5 — Telegram-интеграция (Telethon + Aiogram)

## Основная информация

| Поле | Значение |
|------|----------|
| ID | E5 |
| Статус | Бэклог |
| Зависит от | E4 (получает черновики из Response Pipeline) |
| Необходим для | — (финальный слой, доставляет результат пользователю) |

## Описание

Два параллельных компонента, работающих одновременно:

**MTProto-клиент (Telethon)** — авторизуется в аккаунте пользователя через session-файл, слушает входящие сообщения во всех чатах из whitelist, передаёт их в E4 (Response Pipeline), и после подтверждения владельца отправляет итоговый ответ от имени пользователя.

**Управляющий Telegram-бот (Aiogram)** — отдельный бот, через который владелец взаимодействует с ассистентом: получает черновики с кнопками «Отправить / Редактировать / Отклонить», управляет whitelist'ом (добавляет чаты, переключает режим `auto`/`review`), пополняет память в свободной форме, получает уведомления о чувствительных сообщениях и сбоях.

## Что входит в эпик

- **Авторизация Telethon** — первичная авторизация с кодом/2FA, сохранение session-файла.
- **Event handler `NewMessage`** — фильтрует по whitelist, передаёт сообщение в pipeline E4.
- **Whitelist CRUD** — команды бота для добавления/удаления чатов, просмотра режимов.
- **Доставка черновика** — Aiogram отправляет черновик владельцу с inline-клавиатурой (3 кнопки).
- **FSM редактирования** — состояние «жду исправленный текст» после нажатия «Редактировать».
- **Отправка ответа** — после подтверждения Aiogram-бот говорит Telethon-клиенту отправить текст в исходный чат.
- **Уведомления** — о чувствительных сообщениях, о низкой уверенности (low_confidence), о деградации LLM API.

## Режимы работы чатов

| Режим | Поведение |
|-------|-----------|
| `review` | Все черновики требуют ручного подтверждения |
| `auto` | Черновики отправляются автоматически (кроме чувствительных) |

## Критерий готовности эпика

- [ ] Входящее сообщение из whitelist-чата → черновик появляется в управляющем боте < 10 сек
- [ ] Кнопка «Отправить» — ответ уходит от имени пользователя в исходный чат
- [ ] Кнопка «Редактировать» — бот ждёт новый текст, затем отправляет исправленный
- [ ] Кнопка «Отклонить» — черновик помечается `rejected`, ответ не отправляется
- [ ] Режим `auto` не отправляет автоматически сообщения с флагом `requires_confirmation`
