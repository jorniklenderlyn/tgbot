# Эпик 2 — Профиль стиля (Persona Extraction)

## Основная информация

| Поле | Значение |
|------|----------|
| ID | E2 |
| Статус | Бэклог |
| Зависит от | E1 (нужны обработанные данные переписки) |
| Необходим для | E4 (persona используется в промпте) |

## Описание

CLI-инструмент, который анализирует историю переписки пользователя и формирует JSON-профиль стиля (`persona.json`). Профиль используется как системная часть промпта при каждой генерации ответа — он говорит LLM «пиши как этот человек».

Анализ двухэтапный: сначала детерминированные признаки (считаются без LLM, быстро и воспроизводимо), затем LLM-интерпретация речевых паттернов.

## Что входит в эпик

- **Детерминированный анализатор** (regex, без LLM):
  - средняя длина сообщения
  - доля сообщений с эмодзи
  - частота знаков пунктуации (восклицательный, многоточие)
  - доля заглавных букв
  - список топ-20 слов (без стоп-слов)
- **LLM-интерпретатор** — отправляет в LLM 50 случайных сообщений пользователя (анонимизированных) и просит описать речевые паттерны, типичные обороты, тон.
- **Сборщик `persona.json`** — объединяет оба результата в один файл.
- **CLI-команда `extract-persona <path>`** — запускает весь pipeline, сохраняет `persona.json`.

## Формат persona.json

```json
{
  "style_summary": "Пишет коротко, часто без знаков препинания. Тон неформальный.",
  "avg_message_length": 12,
  "emoji_ratio": 0.15,
  "phrases": ["ок", "понял", "давай", "потом разберёмся"],
  "punctuation_style": "минимум знаков, иногда многоточие",
  "tone": "дружеский, лаконичный"
}
```

## Критерий готовности эпика

- [ ] `extract-persona` принимает путь к JSON-выгрузке и создаёт `persona.json`
- [ ] Детерминированные признаки вычисляются без обращения к LLM
- [ ] В LLM уходят только анонимизированные сообщения
- [ ] `persona.json` валидируется через Pydantic-схему
