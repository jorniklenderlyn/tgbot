## Описание

Реализовать `MessageQueue` в файле `pipeline/message_queue.py` — очередь входящих сообщений на базе SQLite. Решает проблему потери сообщений при временной недоступности LLM или Qdrant.

Логика работы:
1. При получении сообщения из Telethon оно **сразу** попадает в очередь (`status=queued`).
2. Pipeline пытается обработать его немедленно.
3. Если обработка успешна — `status=done`.
4. Если `LLMUnavailableError` — `status=failed`, `retry_count += 1`, `next_retry_at = now + backoff`.
5. Фоновый воркер каждые 30 сек проверяет сообщения с `next_retry_at <= now` и `retry_count < 5` — повторяет обработку.
6. После 5 неудачных попыток — `status=dead`, владелец получает уведомление.

Максимальное время ожидания retry — 5 минут (задержки backoff: 15 сек → 30 → 60 → 120 → 300).

## Что нужно разработать

- Enum `QueueStatus`: `queued`, `done`, `failed`, `dead`
- Класс `MessageQueue` с методами: `enqueue()`, `mark_done()`, `mark_failed()`, `get_ready_for_retry()`
- Фоновый воркер `RetryWorker` (asyncio-задача или отдельный поток), который периодически забирает и повторно обрабатывает сообщения из очереди

## Схема таблицы `message_queue`

| Поле | Тип | Описание |
|------|-----|----------|
| `msg_id` | TEXT (UUID) | Первичный ключ |
| `chat_id` | INTEGER | ID чата |
| `text` | TEXT | Текст входящего сообщения |
| `status` | TEXT | queued / done / failed / dead |
| `retry_count` | INTEGER | Количество попыток |
| `next_retry_at` | TEXT | ISO 8601, когда следующая попытка |
| `received_at` | TEXT | Время получения сообщения |

## Критерий приёмки

- `enqueue(chat_id, text)` — добавляет сообщение в очередь, возвращает `msg_id`
- `mark_done(msg_id)` — помечает как успешно обработанное
- `mark_failed(msg_id)` — увеличивает `retry_count`, вычисляет `next_retry_at` через backoff
- `get_ready_for_retry()` — возвращает сообщения где `status=failed` и `next_retry_at <= now` и `retry_count < 5`
- При `retry_count == 5` — `status=dead`, callback для уведомления владельца
- Черновик не теряется при перезапуске системы (все данные в SQLite)
