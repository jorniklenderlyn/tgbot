# Эпик 1 — Сбор и обработка данных

## Основная информация

| Поле | Значение |
|------|----------|
| ID | E1 |
| Статус | Бэклог |
| Зависит от | — |
| Необходим для | E2 (Persona Extraction), E4 (Response Pipeline) |

## Описание

Всё, что связано с подготовкой обучающих данных для ассистента: импорт JSON-выгрузок из Telegram Desktop, анонимизация переписки перед отправкой в облачный LLM, извлечение пар «входящее сообщение → ответ пользователя» и построение векторного few-shot индекса в Qdrant.

Без этого эпика невозможна работа модулей генерации ответов (E4) и извлечения профиля стиля (E2) — оба используют индексированные пары как источник данных.

## Что входит в эпик

- **Парсер JSON-выгрузки Telegram** — читает файл экспорта (`result.json`), фильтрует сообщения по отправителю, выделяет пары «входящее → ответ».
- **Анонимизатор** — заменяет имена, даты, телефоны, адреса на плейсхолдеры (`{NAME_1}`, `{DATE_1}`, `{PHONE_1}`) через regex перед любой передачей данных в облако.
- **Векторизатор** — кодирует входящие сообщения через `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`.
- **Загрузчик в Qdrant** — создаёт коллекцию `fewshot_pairs`, загружает векторы с payload `{incoming, reply}`.
- **CLI-команда `index-history`** — объединяет все шаги выше в одну команду запуска.

## Критерий готовности эпика

- [ ] CLI-команда `index-history <path>` успешно обрабатывает выгрузку от 1 000 пар
- [ ] Ни одно имя, дата или телефон не попадает в Qdrant без анонимизации
- [ ] Qdrant-коллекция `fewshot_pairs` содержит векторы и payload для каждой пары
- [ ] Время индексации 5 000 пар < 5 минут
