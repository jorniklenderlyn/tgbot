# Задача 4.5 — DraftManager: хранение черновиков в SQLite

## Основная информация

| Поле | Значение |
|------|----------|
| ID | 4.5 |
| Эпик | E4 — Конвейер генерации ответов |
| Статус | Бэклог |
| Зависит от | **4.4** (LLMClient → текст черновика и метаданные) |
| Влияет на | **E5** (Aiogram-бот читает черновики для доставки владельцу) |
| Тип связи с эпиком | Влияет на E4 |

## Описание

Реализовать `DraftManager` в файле `pipeline/draft_manager.py` — хранилище черновиков на базе SQLite. Каждый черновик — это запись, которая хранит входящее сообщение, сгенерированный ответ, флаги безопасности и текущий статус.

`DraftManager` — связующее звено между `LLMClient` (задача 4.4) и `E5` (Aiogram-бот): pipeline сохраняет черновик после генерации, бот читает `pending`-черновики и доставляет владельцу. После действия владельца (отправил/отклонил/редактировал) бот обновляет статус через `update_status()`.

SQLite выбран для надёжности: черновики не теряются при перезапуске системы.

## Что нужно разработать

- Датакласс `Draft` со всеми полями (см. схему ниже)
- Класс `DraftManager` с методами: `save()`, `get()`, `update_status()`, `get_pending()`
- Автоматическое создание таблицы при первом запуске (`CREATE TABLE IF NOT EXISTS`)
- Enum `DraftStatus` с состояниями: `pending`, `sent`, `edited`, `rejected`

## Схема таблицы `drafts`

| Поле | Тип | Описание |
|------|-----|----------|
| `draft_id` | TEXT (UUID) | Первичный ключ |
| `chat_id` | INTEGER | ID исходного чата (Telegram) |
| `incoming_text` | TEXT | Текст входящего сообщения |
| `draft_text` | TEXT | Сгенерированный черновик |
| `requires_confirmation` | INTEGER (bool) | 1 если чувствительная тема |
| `low_confidence` | INTEGER (bool) | 1 если низкая уверенность |
| `status` | TEXT | pending / sent / edited / rejected |
| `created_at` | TEXT (ISO 8601) | Время создания |

## Критерий приёмки

- `save(draft)` — сохраняет черновик, возвращает `draft_id`
- `get(draft_id)` — возвращает `Draft` или `None`
- `update_status(draft_id, status)` — обновляет статус черновика
- `get_pending()` — возвращает только черновики со статусом `pending`
- Черновик не теряется при перезапуске (данные в файле `drafts.db`)
- `requires_confirmation` проставляется из `ClassificationResult.is_sensitive`
- Таблица создаётся автоматически при инициализации класса

## Примерное решение

```python
# pipeline/draft_manager.py
import sqlite3
import uuid
from dataclasses import dataclass
from datetime import datetime, timezone
from enum import StrEnum
from pathlib import Path


class DraftStatus(StrEnum):
    PENDING = "pending"
    SENT = "sent"
    EDITED = "edited"
    REJECTED = "rejected"


@dataclass
class Draft:
    draft_id: str
    chat_id: int
    incoming_text: str
    draft_text: str
    requires_confirmation: bool
    low_confidence: bool
    status: DraftStatus
    created_at: str


CREATE_TABLE_SQL = """
CREATE TABLE IF NOT EXISTS drafts (
    draft_id TEXT PRIMARY KEY,
    chat_id INTEGER NOT NULL,
    incoming_text TEXT NOT NULL,
    draft_text TEXT NOT NULL,
    requires_confirmation INTEGER NOT NULL DEFAULT 0,
    low_confidence INTEGER NOT NULL DEFAULT 0,
    status TEXT NOT NULL DEFAULT 'pending',
    created_at TEXT NOT NULL
)
"""


class DraftManager:
    def __init__(self, db_path: str = "drafts.db"):
        self._db = Path(db_path)
        with self._connect() as conn:
            conn.execute(CREATE_TABLE_SQL)

    def _connect(self) -> sqlite3.Connection:
        return sqlite3.connect(self._db)

    def save(
        self,
        chat_id: int,
        incoming_text: str,
        draft_text: str,
        requires_confirmation: bool = False,
        low_confidence: bool = False,
    ) -> str:
        draft_id = str(uuid.uuid4())
        created_at = datetime.now(timezone.utc).isoformat()
        with self._connect() as conn:
            conn.execute(
                "INSERT INTO drafts VALUES (?,?,?,?,?,?,?,?)",
                (draft_id, chat_id, incoming_text, draft_text,
                 int(requires_confirmation), int(low_confidence),
                 DraftStatus.PENDING, created_at),
            )
        return draft_id

    def get(self, draft_id: str) -> Draft | None:
        with self._connect() as conn:
            row = conn.execute(
                "SELECT * FROM drafts WHERE draft_id = ?", (draft_id,)
            ).fetchone()
        if not row:
            return None
        return Draft(*row[:6], DraftStatus(row[6]), row[7])

    def update_status(self, draft_id: str, status: DraftStatus) -> None:
        with self._connect() as conn:
            conn.execute(
                "UPDATE drafts SET status = ? WHERE draft_id = ?",
                (status, draft_id),
            )

    def get_pending(self) -> list[Draft]:
        with self._connect() as conn:
            rows = conn.execute(
                "SELECT * FROM drafts WHERE status = 'pending'"
            ).fetchall()
        return [Draft(*r[:6], DraftStatus(r[6]), r[7]) for r in rows]
```
