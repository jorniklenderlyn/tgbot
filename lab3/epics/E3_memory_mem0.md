## Описание

Система долговременной памяти ассистента — полностью локальная, без передачи данных в облако. Используется библиотека **Mem0** в self-hosted режиме поверх трёх хранилищ: Qdrant (векторный поиск по смыслу), Neo4j (граф связей между людьми и событиями), SQLite (быстрый доступ к структурированным фактам).

Память хранит: факты о собеседниках (кто такой, о чём договаривались), факты о самом пользователе (планы, предпочтения, расписание), историю прошлых диалогов. При генерации ответа система делает `mem0.search()` и добавляет найденные воспоминания в промпт.

## Что входит в эпик

- **Docker Compose** — разворачивает Qdrant и Neo4j локально; конфигурация через `.env`.
- **Инициализация Mem0** — настройка клиента с указанием локальных endpoint'ов Qdrant и Neo4j.
- **Knowledge Intake модуль** — принимает свободный текст от пользователя (через управляющий бот), передаёт в LLM для извлечения структурированных фактов, сохраняет через `mem0.add()`.
- **Автоматическое пополнение** — после каждого отправленного ответа LLM извлекает новые факты из диалога и добавляет в Mem0.
- **Поиск воспоминаний** — при входящем сообщении вызывает `mem0.search(query, user_id)` и возвращает топ-5 релевантных фактов.

## Критерий готовности эпика

- [ ] `docker compose up` поднимает Qdrant и Neo4j без ошибок
- [ ] `mem0.add("Саша живёт в Москве", user_id="contact_123")` сохраняется
- [ ] `mem0.search("где живёт Саша")` возвращает сохранённый факт
- [ ] Все данные остаются на устройстве пользователя (нет запросов к облачному Mem0 API)
- [ ] Knowledge Intake корректно парсит свободный текст и сохраняет факты
