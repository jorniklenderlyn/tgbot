# Задача 4.1 — MessageClassifier: детекция чувствительных тем

## Основная информация

| Поле | Значение |
|------|----------|
| ID | 4.1 |
| Эпик | E4 — Конвейер генерации ответов |
| Статус | Бэклог |
| Зависит от | — |
| Влияет на | **4.3** (PromptBuilder использует результат классификации) |
| Тип связи с эпиком | Влияет на E4 |

## Описание

Реализовать модуль `MessageClassifier` в файле `pipeline/classifier.py`. Модуль анализирует входящее сообщение и определяет, содержит ли оно чувствительные темы: финансовые обязательства, конкретные даты/время встреч, обещания и договорённости.

Если хотя бы одна чувствительная категория найдена — результат `is_sensitive=True`. Этот флаг передаётся в `PromptBuilder` (задача 4.3) и далее в `DraftManager` (задача 4.5), где черновик получает `requires_confirmation=True` и **никогда** не отправляется автоматически — независимо от режима чата.

Логика двухуровневая:
1. Быстрая regex-проверка — покрывает 90% случаев без вызова LLM.
2. LLM-fallback — включается только если regex вернул неопределённый результат (пограничный случай).

## Что нужно разработать

- Датакласс `ClassificationResult(is_sensitive: bool, categories: list[str])`
- Словарь regex-паттернов `PATTERNS` с тремя категориями: `finance`, `commitment`, `datetime`
- Функцию `classify(text: str) -> ClassificationResult`
- (Опционально) функцию `_llm_classify(text: str) -> bool` для fallback

## Критерий приёмки

- `classify()` возвращает `ClassificationResult` с корректными полями
- Regex покрывает: суммы денег (500 руб, $200, 1к, тысяч), даты (завтра, в пятницу, 15 июня, 15/06), время (в 18:00, в шесть), слова-обязательства (обещаю, договорились, буду, приду, созвонимся)
- При `is_sensitive=True` поле `categories` содержит список найденных категорий
- Проходит ручной тест на 20 примерах: 10 чувствительных (все `True`) + 10 нейтральных (все `False`)
- Функция не обращается к LLM при явном совпадении regex

## Примерное решение

```python
# pipeline/classifier.py
import re
from dataclasses import dataclass, field

PATTERNS: dict[str, list[str]] = {
    "finance": [
        r"\b\d[\d\s]*(?:руб|рублей|₽|\$|баксов|тысяч|тыс|к)\b",
        r"\b(?:переведи|скинь|оплати|должен|верну|скину|кинь на карту)\b",
    ],
    "commitment": [
        r"\b(?:обещаю|обещаем|договорились|буду|приду|приеду|созвонимся|встретимся)\b",
        r"\b(?:ок[,.]?\s+буду|точно|100%|обязательно|не забуду)\b",
    ],
    "datetime": [
        r"\b(?:завтра|послезавтра|сегодня вечером|в\s+(?:понедельник|вторник|среду|четверг|пятницу|субботу|воскресенье))\b",
        r"\b\d{1,2}[./]\d{1,2}(?:[./]\d{2,4})?\b",
        r"\bв\s+\d{1,2}[:.]\d{2}\b",
        r"\bв\s+(?:час|два|три|четыре|пять|шесть|семь|восемь|девять|десять|одиннадцать|двенадцать)\b",
    ],
}


@dataclass
class ClassificationResult:
    is_sensitive: bool
    categories: list[str] = field(default_factory=list)


def classify(text: str) -> ClassificationResult:
    """Классифицирует текст на наличие чувствительных тем."""
    text_lower = text.lower()
    found_categories = []

    for category, patterns in PATTERNS.items():
        if any(re.search(pattern, text_lower) for pattern in patterns):
            found_categories.append(category)

    if found_categories:
        return ClassificationResult(is_sensitive=True, categories=found_categories)

    # Опциональный LLM-fallback для пограничных случаев
    # if _llm_classify(text):
    #     return ClassificationResult(is_sensitive=True, categories=["llm_detected"])

    return ClassificationResult(is_sensitive=False)
```
